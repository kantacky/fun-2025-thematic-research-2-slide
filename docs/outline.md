# LLM によるフォトウォークデータの生成とフォトスポットと撮影者の同時クラスタリング / LLM-Based Photowalk Data Generation and Co-Clustering of Photo Spots and Photographers

キーワード / Keywords: 人工データ生成，行動分析，スポット推薦 / Synthetic Data Generation, Behavior Analysis, Spot Recommendation

## 発表スライド構成（15分発表用）

### アウトライン

- 1. タイトルスライド（1枚）
  - 発表タイトル: フォトウォークにおけるフォトスポットと撮影者の同時クラスタリング
  - 著者名，所属
  - 情報処理学会第88回全国大会

- 2. 背景と動機（2枚）
  - フォトウォークとは何か
    - 散歩中に興味を引いたものを撮影する行為
    - 近年の注目度の高まり
  - フォトウォーク体験向上の重要性
    - ユーザの嗜好に合った新しい視点の提供
    - フォトスポット推薦の可能性

- 3. 問題設定（1枚）
  - フォトウォークデータ収集の困難さ
    - 収集コストが高い
    - プライバシーの観点から大規模データセットが不足
    - SNSデータはノイズが多く，意図や移動経路が不明確

- 4. 関連研究（1枚）
  - LLMを用いた移動軌跡生成の先行研究
    - Li らの構造化出力を活用した空間データ生成フレームワーク
  - 既存研究の限界
    - 一般的な空間データや都市レベルの移動統計が対象
    - フォトウォーク特有の制約（周回性など）への適用は未検証

- 5. 本研究の目的と独自性（1枚）
  - 目的: LLMを活用したフォトウォークデータ生成手法の提案
  - 独自性
    - 被写体カテゴリという意味的情報を含む
    - 出発地点と到着地点が同一という地理的・行動的制約を満たす

- 6. 提案手法: データスキーマ定義（1枚）
  - Session と Record の2つのデータモデル
    - Session: セッションID，ユーザーID，Recordのリスト
    - Record: レコードID，撮影時刻，緯度・経度，被写体カテゴリ
  - 各フィールドに詳細な説明を付与

- 7. 提案手法: プロンプト設計（1〜2枚）
  - 地理的整合性の確保
    - 撮影場所が道路上であること
    - 特定エリア内での行動生成
  - 行動の完結性とリアリティ
    - 出発地点と到着地点が同一
    - 適切な移動時間と撮影枚数の目安
  - エージェントの役割定義
    - 被写体カテゴリの多様性確保

- 8. 実装とモデル選定（1枚）
  - Python と Google GenAI SDK による実装
  - 並列処理による高速な大規模データ生成
  - モデル選定の経緯
    - Gemini 2.5 Flash / Lite の問題点
    - Gemini 3 Flash Preview の採用理由

- 9. 実験設定（1枚）
  - 対象エリア: 北海道函館市（美原・神山・富岡町・中道エリア）
  - 条件
    - 徒歩30分程度
    - 10〜30枚程度の画像撮影
  - 生成規模: 10人 × 5セッション

- 10. 結果（1〜2枚）
  - 生成データの可視化（マップ上へのプロット）
  - 観察された特徴
    - 指定エリア内に概ね分布
    - 道路網に沿った移動軌跡
    - 出発地点と到着地点が近接（周回性の再現）
    - セッションごとの多様性

- 11. 考察（1枚）
  - 地理的整合性の評価
    - 道路中心線からの逸脱やエリア境界付近の揺らぎは軽微
    - 概略経路としては十分な整合性
  - LLMによるエージェント行動の模倣
    - 単調な最短経路移動ではなく探索的行動を再現

- 12. まとめと今後の課題（1枚）
  - まとめ
    - Gemini APIの構造化出力を利用した生成手法を提案
    - 大規模かつ整合性の高いデータ生成が可能
  - 今後の課題
    - フォトスポットと撮影者の同時クラスタリングの実装と評価
    - 実データとの比較による生成データの有用性検証
    - 観光分野の同時クラスタリング手法のフォトウォークへの応用
